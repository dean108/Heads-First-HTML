    Main Points

    • History of HTML
      • HTML 1.0-2.0: very simple
      • HTML 3 1991: browser wars between Netscape/Microsoft
      • HTML 4 1995: w3c standard, separates structure and presentation
      • HTML 4.1 1998: minor changes
      • XHTML 1.0 1999: introduces extensibility and strictness of XML
    • Document type definition - tells browser that file is HTML document
    • W3C validator
    • <img> elements require an alt attribute
    • Adding <meta> tag to specify content type and character encoding
    • transitional vs strict html
      • inline elements (like <img> must be nested inside block elements, not directly in body
      • not just any block element - can't put img element inside
      • Strict HTML rules
        • <html> element needs to go right after DOCTYPE
        • need <head> and <body> elements
        • <head> needs a <title>
        • Only block elements can go directly in <body>
        • block elements can't go inside inline elements
        • no block elements in <p> element
        • <li> elements should only be used in <ul/ol> elements
        • anything can go in a <li> list item
        • Only block elments should go directly in <blockquotes>
        • careful with inline elements
          • never next an <a> element in another <a> element
          • can't nest anything in empty elements such as <img>
        • <alt> tag is required for <img> elements

      Chapter 7 - Moving to XHTML: Putting an 'X' into HTML

      Main Points

      • XHTML - extensible HTML - takes future growth into consideration
      • What is XML?
        • create root element other than html
        • create lots of other elements
        • empty elements must have a forward slash at the end
      • XHTML
        • Change DOCTYPE to XHTML 1.0
        • add xmlns, lang, and xml:lang atributes
      • XHTML is XML - so has interesting advantages over HTML
        • strict syntax allows screen readers to more easily consume web content
        • future proofing
        • can be extended to include additional markup such as vector graphics and mathematical formulas
        • xml is more easily transformed into XHTML than HTML
        • XHTML can be read by apps that read XML
      • XHTML 1.0 Checklist
        • Change DOCTYPE TO Strict or Transitional XHTML
        • Add xmlns, lang, and xml:lang attributes to the <html> opening tag
        • <html> tag must be first tag after DOCTYPE and the </html> closing tag must be last tag in document
        • All element names must be in lower case
        • Opening tags must have closing tags, or in the case of empty elments, the tag must end with a space and then />
        • All attributes must have values, and those values must be surrounded by double quotes
        • &amp must be used for &, and other special characters must be generated similarly
      • xmlns attribute - a unique identifier that informs the browser which particular xml language is being used
      • The root element is <html> rather than <xhtml> because XHTML is meant to be backward compatible

      Chapter 9 - Styling with Fonts and Colors: Expanding your Vocabulary

      Main Points

      • font properties
        • font-family - fonts divided into font families. Only certain fonts installed on computers
        • Font-size
        • color - sets the font color
        • font-weight - lighter, normal, bold, bolder
        • text-decoration - underline, overline, line-through
      • font-family
        • set of fonts that share characteristics
        • 5 families: sans-serif, serif, monospace, cursive, and fantasy
        • Serif has look associated with newspaper print - serifs are the decorative barbs and hooks on the ends of letters
        • sans-seriffonts without serifs. Considered more readable on computer screens than serif
        • monospace> characters have constant width - 'i' takes up same space as 'm' - primarily usd to show code snippets
        • Cursiveincludes fonts that look handwritten. Sometimes used in headings
        • Fantasynot widely available and rarely used for serious web designs
        • font stacks - list of fonts in order of preference belonging to the family
        • need to make sure that you need to include fots that are likely to be on the various platforms (mac, windows, linux, ios, etc) that folks will be using your application or website
      • font sizes
        • specify in: 1)pixels - px, 2)relative - 100%, 3) em - relative unit - so 1.2 = 120%
        • keywords
          • Each size is about 20% larger than the other
          • small is usually 12 pixels in height
          • keyword size can vary across browsers, and users can redefine them
        • How to define sizes
          • choose keyword (small or medium), and define as font size in body
          • specify other fonts relative to that size in either percentage or em
        • font-weight
          • can use bold, normal
          • bolder or lighter not usually used
          • can also set with number between 100 and 900 - not generally used
        • font-style
          • italic - not all fonts support italic - rather than a specifically designed slanted set of characters, browser applies slant to normal characters
          • oblique - regular characters slanted to the right
          • can just choose italic, and browser will figure out if it italic characters or just slants
        • web colors
          • specified in terms of how much red, green, and blue make up the color - each from 0 to 100
          • css only defines 17 colors
          • you can specify web colors using: 1) name, 2) relative percentages of r, g, and b, and 3) hex codes
          • hex code calculation: 1) separate hex color into 3 components, 2) convert hex number into decimal: (first number *16) + (second number) - cd = (12 * 16) + 13
        • text decoration to create underline, etc

        Chapter 10 - The Box Model: Getting Intimate with Elements

        Main Points

        • line-height property
          • specifies the amount of vertical space between each line of text
          • Can improve readability, and contrast different sections of the site
          • Can be set with px, %, and em
        • The Box Model
          • Content area - where content goes, edge is where padding starts
          • Padding - between content and border
          • Border - outside of element
          • Margin - between border and other elements
          • css allows you to customize aspects of each of these things
        • background-image property
          • sets a background image of an element
          • not used to place images in a page, that is what <img> tags are for
          • background-image is for pure presentation, <img> element is where image has more substantial role
          • background-repeat: can have value of repeat, no-repeat, repeat-x, repeat-y, and inherit
          • background-position - positions the image
        • Borders
          • border-syle: solid, dotted, dashed, etc
          • border-width: keywords or pixels
          • border-color
          • Also border-top-width
        • id selector
          • select by using # character in front of id name
          • can select element#id - p#footer
          • id selector only matches one element in a page
          Using multiple style sheets
          • priority goes from top to bottom, with style sheets on bottom taking precedence

          Chapter 11 - Divs and Spans: Advanced Web Construction

          Main Points

          • <div>s
            • divides elements into logical sections - containers
            • divs can be nested within divs
            • divs should be used to separate into logical sections for clarity and styling, not just to create a lot of unnecessary structure
            • divs can be styled with css
            • the width property of divs specifies content width only, the width of the left/right margins, left/right padding, + border width(x2) must be added to content width of the box
            • Can specify width in pixels or percent
            • setting text-align property for div will align all in-line elements
            • text-align should only be set on block elements - has no effect on inline elments
          • Descendant Selectors - parent descendant {
          • CSS Shorthand
            • example margin: 0px 20px 30px 10px; (clockwise top right bottom left)
            • margin: 20px; (sets 20px margin for all)
            • margin: 20px 30px; (sets top/bottom at 20px, and right/left at 30px)
            • background: white url(images/image.gif) repeat-x; (does background-color, background-image, and background-repeat)
            • font: font-style font variant font-width font-size/line-height font family
          • Spans
            • allows one to group inline characters
            • example: <li><span class="artist" >Grateful Dead</span>, <span class = "song">Box of Rain</span></li>
            • Style spans just like divs
          • pseudo-classes
            • acts like a class, but wasn't explicitly defined
            • example - a:link a:visted a:hover
          • Cascading Style Sheets
            • there are multiple style sheets: 1) one's that author writes, 2) ones that the reader's create, and 3) browser's default styles
            • to figure out style for a particular element:
              1. gather up auther, reader, and browser style sheets
              2. find all the declarations that match
              3. sort all matches in order of author, reader, browser
              4. sort all declarations in order of specificity
              5. sort any conflicint rules in order they appar in their individual style sheets. Take list and order any conflicting rules so that the ones appearing later are more important
              6. first rule in the sorted list wins
            • calculating specificity - look at 3 numbers
              • Does the selector have any ids? One point each
              • Does the selector have any classes or pseudo-classes? One point each
              • Does the selector have any element names> One point each
              • bigger the number, the higher the specificity
